    Project maintained by Tim Vaughan

    MASTER is a software package aimed at providing a general means of creating simulators for stochastic models of structured and unstructured population dynamics. It also supports the simulation of inheritance trees and networks under these same models.

    System Requirements

    MASTER is a Java application, and is thus fundamentally platform-agnostic. In order to use it, v1.6 or later of the Java Runtime Environment is required. This likely exists on your system already, but if not you may obtain it here.

    Installation

    Installation of MASTER is very straight-forward. Simply extract the following archive:

    • MASTER-1.0.zip
    Once complete, the new folder MASTER-1.0/ will contain a number of files. On systems running modern Windows or OS X installations, double-clicking on MASTER-1.0.jar will bring up a file selector that you can use to select an XML input file describing a simulation. On Linux computers, the shell script master_linux.sh can be used to run Master from the command line, with the path to an XML input file given as a command line argument. (In this case, making a symbolic link to master_linux.sh from a directory in your execution path like usr/local/bin/ will allow you to execute MASTER on the command line easily from other locations in your file system.)

    Examples

    As well as the software itself, the archive above contains a folder named examples/ which itself contains example MASTER input files for performing a variety of different kinds of simulations. Note that if you use the graphical file selector to run one of these, the output file(s) generated will be left in the folder containing MASTER-1.0.jar.

    Documentation

    For the full documentation of MASTER, please refer to the wiki. This includes a fuller discussion of the foundational goals of the software, a complete description of the XML input format and a set of tutorials aimed at getting new users up to speed as easily as possible.

    Feedback, Feature Requests and Bug Reports

    Feature requests and bug reports should be submitted via the project's issues page. Any other feedback can be addressed directly to Tim Vaughan.
